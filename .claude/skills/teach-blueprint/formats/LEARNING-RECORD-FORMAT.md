# LEARNING-RECORD-FORMAT.md

Ultra-terse records of demonstrated understanding.

## Format

```markdown
# 0001-slug-of-insight

**Core Insight** (1–3 sentences)

**Status**: active | superseded by 000X
**Evidence**: (how understanding was demonstrated)
**Implications**:
- ...
```

## When to Create
- Genuine demonstrated understanding (not just exposure)
- Prior knowledge disclosure that changes the plan
- Correction of a misconception
- Mission shift that affects future lessons

## Rules
- Number sequentially.
- Supersede, never delete.
- Do **not** use for coverage logs or glossary duplicates.
- Only write when it materially affects the next ZPD decision.
