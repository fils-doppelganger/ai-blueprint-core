# MISSION.md Format

One mission per teaching workspace.

## Required Sections

**Mission Statement**  
A single concrete, real-world goal the learner wants to achieve (e.g., "Help my repository implement the NIAID Blueprint so our datasets are properly indexed in the NIAID Data Ecosystem Discovery Portal").

**Success Criteria** (observable, measurable)
- List 3–5 specific, verifiable outcomes.

**Constraints**
- Time, team size, technical stack, repository maturity, etc.

**Out of Scope** (explicitly declared)

**Last Updated**  
Date + reason for any revision.

## Rules
- Keep under one screen.
- Revise when goals shift; cross-link from learning records.
- If the user cannot articulate a clear mission, interview them before proceeding.
