# Teaching Phases for teach-blueprint

## Phase 0: Mission & Workspace Setup
- Confirm or create `MISSION.md` (`formats/MISSION-FORMAT.md`, `MISSION-EXAMPLE.md`)
- Establish `RESOURCES.md` from `formats/RESOURCES-FORMAT.md` (bootstrap list) and initial `GLOSSARY.md`
- Create `reference/` when first cheat sheet is needed (`reference/README.md`)
- Determine learner role(s)

## Phase 1: Knowledge Foundation
- Deliver core concepts with minimal difficulty
- Build shared glossary
- Cite all sources from `RESOURCES.md`

## Phase 2: Skills Practice
- Introduce desirable difficulty (quizzes, exercises, retrieval)
- Generate interactive HTML lessons
- Create learning records on demonstrated mastery

## Phase 3: Wisdom & Application
- Surface real-world judgment questions
- Connect to community resources when appropriate
- Support implementation planning (work plans, outreach)

## Phase 4: Review & Progression
- Read latest learning records
- Update mission if needed
- Propose next ZPD-appropriate lesson or capstone

## State Tracking
- Use `learning-records/` to calculate current position in curriculum
- Maintain `NOTES.md` for learner preferences and pacing notes
